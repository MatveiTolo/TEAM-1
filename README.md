# CAESAR - Система управления проектами

## Что это

CAESAR — веб-приложение для управления задачами и багами. Работает как Trello, с Telegram-уведомлениями, ролями и календарём.

---

## Быстрый старт


### 1. Запустить клиент

cd caesar.client
npm install
npm run dev
Открыть в браузере: `https://localhost:51203`


### 2. Запустить сервер (в отдельном терминале)

cd CAESAR.Server
dotnet run

Сервер запустится на `http://localhost:5254`

---

## Быстрый вход (для тестирования)

На странице входа нажми одну из кнопок:

- **Войти как admin** — Главный администратор
- **Войти как developer** — Разработчик
- **Войти как tester** — Тестировщик
- **Войти как viewer** — Наблюдатель

---

## Что уже работает

- Страница входа / регистрации
- Создание проекта с выбором темы
- Список проектов
- Доска с 4 колонками: Преподготовка → Выполнение → Тестирование → Готово
- Карточки задач с названием, ID, дедлайном, исполнителем
- Drag-and-Drop (перетаскивание карточек между колонками)
- Подсветка дедлайнов (красный — просрочено, жёлтый — завтра)
- Создание задач через форму
- Страница задачи с историей изменений
- Профиль пользователя
- Панель супер-админа (управление пользователями и проектами)
- Адаптивность (работает на ПК, планшете и телефоне)

- Мок-режим (работает без бэкенда)

---

## Мок-режим

Проект работает на фейковых данных, поэтому бэкенд пока не нужен.  
В правом нижнем углу экрана есть бейдж: `🧪 МОК-РЕЖИМ`

Когда бэкенд будет готов — переключение займёт одну строчку в коде.

---

## Требования

- Node.js v18+
- .NET SDK v8+
- Git

---

## Переключение на реальный бэкенд

Когда бэкенд будет готов:

1. Открой `src/services/mockApi.ts`
2. Найди строчку: `const USE_MOCK = true;`
3. Поменяй на: `const USE_MOCK = false;`

Готово! Фронт начнёт работать с реальным API.
